## 배경: 왜 ReadMate인가?

### 문제 정의

**시각장애인의 학습환경 현실:**
- **84.6%의 점자 문맹률** (보건복지부통계청, 2024)
- 기존 AI 서비스는 **단순 스크린리더(음성), 녹음, 단순 TTS**만 제공
- **학습 구조와 이해도 향상에 초점을 둔 AI 없음**
- 교과서 변환에 시간과 노력 필요

### 기회

AI 기술의 발전으로 **멀티모달 처리 가능**  
시각장애인을 위한 **실제 필요(pain point) 존재**  
교육 + 접근성 결합 = **높은 사회적 임팩트**

## 솔루션: ReadMate의 3가지 핵심 기능

### 01 **Vision to Voice** (시각 → 음성)
PDF나 이미지 학습 자료를 음성으로 변환
- OCR(Qwen 2.5-VL-7B)로 텍스트 추출
- STT(Whisper Medium)로 정확한 인식

### 02 **Audio to Insight** (음성 → 이해)
추출된 학습 내용을 LLM으로 분석해 핵심 요약 제공
- LLM(GPT-4.1-mini)으로 요약 및 핵심 파악
- 3-5문장의 간결한 핵심 포인트 생성

### 03 **Interactive Q&A** (양방향 질의응답)
학생들이 음성으로 질문하고 AI가 답변
- LLM 기반 질의응답
- 복습 퀴즈 및 메모 기능

## 🛠️ 기술 스택

### AI/ML
- **STT:** Whisper Medium (OpenAI) - 정확도 96%
- **OCR:** Qwen 2.5-VL-7B - 한국어 최적화
- **LLM:** GPT-4.1-mini - 빠른 추론 속도
- **TTS:** ElevenLabs - 한국어 자연스러운 목소리

### Backend
- **Framework:** FastAPI (Python 3.11)
- **Database:** PostgreSQL
- **Cache:** Redis
- **Async:** asyncio, aiohttp

### Frontend
- **Framework:** React 18 + TypeScript
- **Styling:** Tailwind CSS
- **UI Components:** shadcn/ui
- **State Management:** Zustand

### DevOps
- **Containerization:** Docker
- **CI/CD:** GitHub Actions
- **Deployment:** Vercel/AWS

## 🎨 시각장애인을 위한 고려사항

### 접근성 설계

✅ **음성 브라우저 자동 활성화**
- 페이지 로드 시 친절한 음성 안내

✅ **다양한 목소리 선택**
- 사용자 맞춤 목소리 설정 (보이스 클로닝)
- 속도 조절 가능 (0.5x ~ 2x)

✅ **기본 서비스 음성과 강의 음성 분리**
- 듀얼 보이스 시스템
- 요약 = 중성 음성, 강의 = 강사 음성

✅ **키보드 네비게이션 완벽 지원**
- 마우스 없이도 모든 기능 접근 가능

✅ **고대비 테마**
- WCAG 2.1 AA 준수
- 저시력 사용자도 사용 가능

## 🎓 핵심 모델 상세

### 1. STT (Speech-to-Text)

**선택 모델: Whisper Medium (OpenAI)**

```yaml
모델명: Whisper Medium
특징:
  - 정확도: 96%
  - 한국어 지원: ✅ (초음 충분히 좋음)
  - 속도: 중간 (GPU 기준 5초 음성 = 1초 처리)
  - 비용: 무료 (오픈소스)

선택 이유:
  ❌ Whisper Small: 음성 인식 오류 많음 (93%)
  ✅ Whisper Medium: 현장 정확도 충분
  ⚠️ Whisper Large: 비용 대비 개선 미미
```

### 2. OCR (Optical Character Recognition)

**선택 모델: Qwen 2.5-VL-7B**

```yaml
모델명: Qwen 2.5-VL-7B (시각 언어 모델)
특징:
  - 한국어 인식 최상
  - PDF와 이미지 모두 지원
  - 텍스트와 표 인식 가능
  - JSON 구조화 출력 가능

선택 이유:
  ❌ PaddleOCR: Windows GPU 환경에서 호환성 문제
               Paddle과 PyTorch의 cuDNN 요구사항 충돌
  ✅ Qwen 2.5-VL-7B: 텍스트 기반 PDF도 이미지도 처리
                      한국어 추출 최상 (가장 중요!)
```

### 3. LLM (Large Language Model)

**선택 모델: GPT-4.1-mini**

```yaml
모델명: GPT-4.1-mini
특징:
  - 응답 속도: 빠름
  - 안정성: 높음
  - 비용: 저렴 (GPT-4o 대비 1/10)

선택 사유:
  ❌ 로컬 모델 (Qwen, Gemma): 추론 속도 느림
                              실시간 서비스에 부적합
  ✅ GPT-4.1-mini: 빠른 속도 + 안정적인 응답
```

### 4. TTS (Text-to-Speech)

**선택 모델: ElevenLabs**

```yaml
모델명: ElevenLabs
특징:
  - 한국어 목소리: 자연스러움 (최상)
  - 보이스 클로닝: 가능
  - 운영 비용: 증가 (단점)

선택 이유:
  ❌ Zonos: 한국어 품질 낮음
  ❌ Edge TTS: 보이스 제한 + 클론 불가
  ❌ Kokoro MMS: 한국어 지원 미숙
  ✅ ElevenLabs: 한국어 품질 최고, 사용자 만족도 높음
```
